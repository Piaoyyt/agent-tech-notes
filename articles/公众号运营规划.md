## 一、公众号定位

### 1.1 基本信息

| 项目 | 内容 |
|------|------|

### 1.2 核心价值主张

三个关键词定义这个号的独特性：

- **实战经验**：来自物流行业 Agent 落地的第一手经验，不是纸上谈兵
- **系统思维**：从单点技术到多 Agent 协作架构的体系化思考
- **前沿洞察**：追踪 Agent 技术趋势、论文解读、开源项目测评

### 1.3 和其他号的差异化

> 写这段是为了提醒自己：不要写成"又一个 AI 资讯搬运号"

| 常见技术号的做法 | 本号的差异化 |
|-----------------|-------------|
| 转载/搬运 AI 新闻 | 只发原创，每篇都有自己的观点和实践 |
| 泛 AI 话题全覆盖 | 聚焦 Agent + 大模型，不做大而全 |
| 纯理论/论文解读 | 理论 + 可运行代码 + 落地经验结合 |
| 不成体系的碎片化内容 | 系列化输出，有知识地图串联 |

### 1.4 目标受众

**主要受众**：
- 算法工程师、大模型应用开发者
- 正在或即将做 Agent 落地的技术团队

**次要受众**：
- 技术管理者（需要理解 Agent 技术选型）
- 产品经理（需要理解 Agent 能做什么、不能做什么）

**潜在受众**：
- 对 AI 技术感兴趣的高年级学生/研究生
- 跨领域想了解 Agent 技术的应用型开发者

## 二、内容支柱（Content Pillars）

> 所有文章选题必须能归入以下四个支柱之一，避免选题跑偏。

### 支柱一：技术深度解析（占比 ~40%）

**定位**：帮助读者理解 Agent 技术的原理和实现方式。

**内容方向**：
- Agent 决策模式原理（ReAct / Reflection / Reflexion / Plan-and-Execute）
- Memory 模块设计（短期记忆 → 长期记忆的演进）
- 工具调用（Function Calling 设计模式、错误处理、工具编排）
- 垂域模型微调（SFT / GRPO 在 Agent 场景的应用）
- Agent 框架源码解析（LangGraph / LangChain / 自研）

**选题标准**：
- ✅ 有完整可运行的代码示例
- ✅ 有自己的实战经验或踩坑记录
- ✅ 能在 Agent 技术全景图中找到对应位置
- ❌ 纯搬运官方文档的不写

### 支柱二：实战案例分享（占比 ~30%）

**定位**：展示 Agent 技术如何在真实业务中落地，让读者看到"怎么用"。

**内容方向**：
- 传统 Workflow 到 A2A 架构的演进路径
- 性能优化与成本控制实战
- Agent 评测体系搭建
- 调试/排查 Agent 系统的经验
- 项目复盘（成功 + 失败案例）

**选题标准**：
- ✅ 所有公司案例必须脱敏处理
- ✅ 要有"问题 → 方案 → 效果"的完整链路
- ✅ 包含可量化的结果（性能提升了 X%、成本降低了 Y%）
- ❌ 纯广告/PR 性质的不写

### 支柱三：前沿洞察与趋势（占比 ~20%）

**定位**：帮读者快速了解 Agent 领域的最新动态和未来方向。

**内容方向**：
- 重要论文精读（选与 Agent 落地强相关的）
- 行业动态与标杆案例分析
- 技术趋势预测
- 开源 Agent 框架/工具深度测评

**选题标准**：
- ✅ 有自己的分析和判断，不只是论文摘要
- ✅ 和 Agent 落地强相关的才写，不追热点
- ❌ 每天都有的 AI 资讯不写

### 支柱四：课程系统化内容（占比 ~10%，后期逐步提升）

**定位**：将碎片化文章串联成体系化知识，为课程输出铺路。

**内容方向**：
- 知识体系梳理（定期发布 Agent 技术地图更新版）
- 学习路径设计
- 系列教程（如"Agent 从入门到精通"）
- 课程预告/反馈收集

**选题标准**：
- ✅ 每篇文章标注在知识体系中的位置
- ✅ 有明确的难度等级标注（入门 / 进阶 / 实战）
- ❌ 没有经过验证的内容不放进来

## 三、内容形式规范

### 3.1 文章类型与字数

| 类型 | 字数 | 适用场景 | 发布频率 |
|------|------|---------|---------|
| 深度长文 | 2000-4000 字 | 技术原理、实战案例 | 核心内容，每周 1-2 篇 |
| 系列教程 | 每期 1500-2500 字 | 系统化知识输出 | 连续 3-5 期为一个系列 |
| 技术笔记 | 800-1500 字 | 论文解读、前沿洞察 | 灵活穿插 |
| 互动问答 | 500-1000 字 | 月度读者问答 | 每月 1 期 |

### 3.2 文章结构模板

每篇文章统一使用以下结构：

```
1. 引子（200 字）
   → 用一个具体的问题/场景/痛点开场，让读者产生共鸣

2. 核心内容（1500-3000 字）
   → 原理解释 + 代码示例 + 图表说明
   → 踩坑记录（如果有的话）

3. 总结与延伸（300 字）
   → 关键点回顾
   → 这篇文章在知识体系中的位置
   → 延伸阅读推荐

4. 互动环节（100 字）
   → 思考题 / 下期预告 / 评论区引导
```

### 3.3 标题公式

| 类型 | 公式 | 示例 |
|------|------|------|
| 实战类 | 《XXX 实战：从 X 到 Y 做 XXX》 | 《Agent 记忆系统实战：从 0 到 1 搭建长期记忆模块》 |
| 解析类 | 《深度解析 XXX：原理、实现与优化》 | 《深度解析 ReAct：Agent 自主决策的核心机制》 |
| 案例类 | 《XXX 落地实录：X 个月实现 XXX》 | 《Agent 系统优化实录：响应时间从 10s 降到 2s》 |
| 洞察类 | 《XXX 趋势/解读：XXX》 | 《2026 Agent 技术趋势：从单 Agent 到多 Agent 协作》 |

### 3.4 代码展示规范

- 所有代码提供**完整可运行**版本（不只贴关键片段）
- 语法高亮，关键步骤加注释
- 配套 GitHub 仓库同步更新
- 复杂项目附 README 说明

## 五、相关设计范式

### 5.1 写作风格

**三个关键词定义品牌声音**：

1. **专业但不冷**：用工程师的语言讲技术，避免过度学术化，适当分享踩坑经历
2. **实战派**：每篇文章都要有"怎么做"，不只是"是什么"
3. **系统化**：强调知识点之间的联系，帮助读者建立全局视野

**禁止的写作习惯**：
- ❌ 不堆砌术语装专业
- ❌ 不写"XXX 是革命性的技术"这类空洞吹捧
- ❌ 不发没有自己观点的搬运内容
- ❌ 不在技术文里卖焦虑

### 5.2 排版规范

- 字号：正文 15px，标题 18-20px
- 行间距：1.6-1.8 倍
- 配色：主色（深蓝/靛蓝）+ 强调色（青色），避免花哨
- 代码块：深色背景 + 语法高亮
- 封面图：统一风格模板（用 Canva/Figma 做一个固定模板）
- 文章头部：统一格式（标题 + 知识地图位置标签 + 难度标签）

### 5.3 封面图规范

- 尺寸：900 × 500 px（2.35:1）
- 风格：深色科技感底图 + 大号标题文字
- 建议：用 Figma 做一个模板，每期只改标题文字即可

## 六、内容生产工作流

### 6.1 标准流程（单篇文章）

```
选题策划（周日晚间 30 分钟）
  → 确定本周要写的 1-2 个选题
  → 分配到对应的 Content Pillar
  → 检查是否在编辑日历中

资料准备（写作前 1-2 天）
  → 整理相关代码 / 案例 / 数据
  → 查阅最新论文和技术文档
  → 画流程图 / 架构图（如果需要）

内容创作（3-5 天）
  → Day 1：写初稿（2000-4000 字）
  → Day 2：验证代码 + 补充截图
  → Day 3：修订 + 配图 + 排版
  → Day 4：预览 + 最后检查
  → Day 5：发布日

发布与推广（发布日）
  → 公众号发布
  → 同步到知乎 / 掘金
  → 技术群分享
  → 朋友圈推广

发布后（1 周内）
  → 回复评论和私信
  → 记录数据（阅读量、分享数、评论数）
  → 收集读者反馈，优化下期内容
```

### 6.2 内容储备策略

- 始终保持 **2-3 篇储备稿**，防止工作忙时断更
- 储备稿不需要完美，但框架和核心内容要完整
- 工作中遇到好的案例/踩坑点，随手记录，不用等"有灵感"才写

### 6.3 内容复用策略

一篇深度长文可以拆解为多个分发物料：

```
深度长文（2000-4000 字，公众号）
  → 精简版（800-1500 字，知乎/掘金）
  → 知识点卡片（配图 + 核心观点，朋友圈/即刻）
  → 代码片段（GitHub + 社群分享）
  → 问答素材（读者提问的回复素材）
```

- 前 5 篇文章重点打磨质量，追求"爆款"潜质

- 在已有技术社群分享（不过度营销，真诚分享）
- 知乎回答相关问题，引流到公众号
- GitHub README 附上公众号信息

**互动策略**：
- 文末设置思考题，引导评论
- 及时回复每条评论，建立互动氛围
- 考虑建一个读者交流群（飞书/企业微信群）

### 7.3 成长期策略（1000-5000 粉丝）

- 启动系列化内容（如"Agent 从入门到精通"）
- 考虑 B 站同步视频内容
- 寻找同领域公众号互推

## 十、风险与应对

| 风险 | 应对策略 |
|------|---------|
| 公司敏感信息泄露 | 所有案例严格脱敏，隐去具体业务数据 |

| 内容质量波动 | 建立写作 checklist，发布前自检 |
| 课程转化不及预期 | 提供免费试学内容，先建立信任再转化 |
